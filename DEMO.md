# CellKit Actions Demo Script

This script is the recommended reviewer/community walkthrough for the browser workflow.

## Goal

Demonstrate the full testnet flow:

```text
build unsigned transaction -> sign externally -> validate -> dry-run -> broadcast -> inspect explorer link
```

## Requirements

- CellKit backend running with CKB testnet RPC/indexer configuration
- CellKit frontend running with `NEXT_PUBLIC_API_URL` pointing to the backend
- A funded CKB testnet address
- Compatible external CKB signing tool or wallet

## Steps

1. Open the frontend.
2. Go to `/playground`.
3. Select `CKB Transfer`.
4. Enter:
   - `fromAddress`
   - `toAddress`
   - `amountCkb`
   - `feeRate`
5. Build the unsigned transaction.
6. Inspect the response summary:
   - selected cell count
   - estimated fee
   - change amount
   - warnings
7. Copy the unsigned transaction JSON.
8. Sign the transaction externally.
9. Go to `/broadcast`.
10. Paste the signed transaction JSON.
11. Click `Validate`.
12. Confirm validation passes.
13. Click `Dry Run`.
14. Confirm CKB RPC returns dry-run cycles.
15. Click `Broadcast`.
16. Confirm CellKit returns:
   - transaction hash
   - testnet explorer URL
17. Open the explorer URL and verify the transaction page loads.

## Expected Result

The demo is successful when:

- The unsigned transaction is generated by CellKit.
- The transaction is signed externally.
- The signed payload passes CellKit validation.
- Dry-run succeeds through CKB RPC.
- Broadcast returns a real CKB testnet transaction hash.
- The returned explorer link opens the transaction page.

## Failure Cases To Show

The broadcast page should also clearly reject:

- Empty witness arrays
- Zeroed placeholder witnesses
- Too-short witness data such as `0x1234`
- Malformed input out points
- Missing backend RPC configuration

These cases demonstrate that CellKit is not blindly broadcasting arbitrary JSON.
