# CellKit Actions Demo Script

Use this as the recording guide for the Spark Program Milestone 3 demo.

## Demo Goal

Show the complete testnet workflow in one clear pass:

```text
build unsigned transaction -> sign externally -> validate -> dry-run -> broadcast -> inspect explorer link
```

The demo should make three things obvious:

- CellKit builds inspectable CKB transaction JSON from action inputs.
- Private keys stay outside CellKit; signing happens externally.
- Signed payloads are validated and dry-run before testnet broadcast.

## Pre-Recording Checklist

Backend:

```bash
cargo fmt -- --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test
cargo run
```

Frontend:

```bash
pnpm lint
pnpm build
pnpm dev
```

Environment:

- Backend has `CKB_RPC_URL` configured for CKB testnet.
- Backend has `CKB_INDEXER_URL` configured for CKB testnet live-cell lookup.
- Frontend has `NEXT_PUBLIC_API_URL` pointing to the backend.
- The sender testnet address is funded.
- External CKB signing tool or wallet is ready.
- Browser zoom is set to 100% and the window is wide enough to show the result panel.

## Recording Flow

1. Open the frontend homepage and briefly show the protocol console UI.
2. Go to `/playground`.
3. Select `CKB Transfer`.
4. Enter:
   - `fromAddress`
   - `toAddress`
   - `amountCkb`
   - `feeRate`
5. Click `Build Transfer`.
6. Show the response summary:
   - selected cell count
   - estimated fee
   - change amount
   - warnings, if any
7. Open the transaction JSON sections and show that the payload is inspectable.
8. Copy the unsigned transaction JSON.
9. Sign the transaction externally.
10. Go to `/broadcast`.
11. Paste the signed transaction JSON.
12. Click `Validate` and show that validation passes.
13. Click `Dry Run` and show returned cycles.
14. Click `Broadcast`.
15. Show the returned transaction hash and testnet explorer URL.
16. Open the explorer URL and confirm the transaction page loads.

## Failure Shots

Record one or two short rejection cases after the happy path:

- Paste the unsigned payload with placeholder witnesses and click `Validate`.
- Paste a witness such as `0x1234` and click `Validate`.
- Remove or corrupt an input `previous_output.tx_hash` and click `Validate`.

These shots demonstrate that CellKit is not blindly broadcasting arbitrary JSON.

## Success Criteria

The demo is complete when the recording shows:

- Unsigned transaction generated by CellKit.
- External signing boundary clearly visible.
- Signed payload validation passing.
- CKB RPC dry-run passing.
- Testnet broadcast returning a real transaction hash.
- Explorer link opening successfully.
